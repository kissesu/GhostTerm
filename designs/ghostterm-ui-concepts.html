GhostTerm / UI Concepts / Non-IDE Directions

不是编译器，不是编辑器壳子，而是三种全新的 AI CLI 桌面工作形态

这不是在传统 IDE 框架上换皮。三套设计都把“文件树-编辑器-终端”降级为局部能力，转而把 GhostTerm 定义成 会话、上下文、执行、修复、切换、记忆共同构成的桌面操作环境。每套都包含 Workspace / Tools / Settings， 但组织逻辑完全不同。
