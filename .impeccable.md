# GhostTerm — Design Context

## Target Audience
Developer power users who prioritize both aesthetics and functionality — Ghostty and Claude Code users who are particular about their tools. macOS-native, keyboard-first workflow.

## Use Cases
- Code editing (CodeMirror 6) with multi-project sessions
- Integrated terminal (xterm.js + PTY over WebSocket)
- Project group management (sidebar accordion)
- Lightweight IDE alternative for CLI-focused developers

## Brand Personality
**"Obsidian Forge"** — A craftsperson's precision instrument. Dark, warm, intentional.
Like a quality mechanical keyboard or the Ghostty terminal itself: functional but refined, confident but not flashy. Not corporate-clean, not hacker-edgy. A tool you're proud to use.

## Tone
- Dense but breathable — power users want density without claustrophobia
- Warm precision — the brass/amber accent grounds the cool dark navy
- Quiet confidence — no gratuitous effects, no decoration for decoration's sake
- Memorable without being theatrical

## Differentiation
The brass accent on near-black navy is already distinctive. No AI-slop cyan/purple.
The typography and depth system need to match the quality of the color choices.

## Technical Constraints
- Tauri desktop app (macOS-first)
- CSS custom properties for theming (dark/light/system)
- Inline styles (no CSS modules, no Tailwind)
- All colors via OKLCH tokens
- Font loaded from Google Fonts (internet required)

## Design Decisions Made
- Font: Outfit (not Inter — too generic)
- Accent: Brass/amber oklch(74% 0.18 56) — keep, it's distinctive
- Active state: Full accent border + warm background tint (no left-only thick bar)
- App identity: Terminal ">" icon + "GhostTerm" in titlebar center
