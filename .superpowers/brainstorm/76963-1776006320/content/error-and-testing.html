GhostTerm MVP -- 错误处理与测试策略

失败场景分析 + 分层测试方案

错误处理：按模块逐一分析

原则：暴露错误、修复根因，不做降级或静默吞没

失败场景 检测方式 处理
pty_manager
PTY 进程启动失败
(shell 不存在、权限不足)		 spawn_pty 返回 Err 前端终端区域显示错误信息 + 重试按钮
WebSocket 连接断开
(进程退出、网络异常)		 WebSocket onclose 事件 终端显示 "连接已断开" + 自动重连（PTY 进程仍存活时）或提示重启
PTY 进程意外退出
(shell exit、crash)		 Rust 端 wait() 检测子进程退出 Event 通知前端 → 终端显示退出码 + "按任意键重启"
fs_backend
文件读取失败
(文件被删、权限、编码)		 read_file 返回 Err 编辑器标签页显示错误状态；如果是已打开的文件被外部删除，标记为 "已删除"
文件保存失败
(磁盘满、只读)		 write_file 返回 Err Toast 通知具体错误原因，保留编辑器中的未保存内容
notify watcher 崩溃 watcher 线程 panic 捕获 Event 通知前端 "文件监听已停止"，提供重启监听按钮
git_backend
非 Git 仓库 git2 open 返回 Err Changes 和 Worktrees 标签页显示 "当前目录非 Git 仓库"，功能禁用但不影响其他模块
git2 worktree API 不可用 调用时返回 Err fallback 到 git CLI；如果 git CLI 也不可用，Worktrees 标签页显示具体错误
project_manager
projects.json 损坏/不可读 serde 反序列化 Err 重置为空列表，日志记录错误。不静默使用默认值

测试策略

分层测试，每层有明确的验证目标

Rust 单元测试

验证后端逻辑正确性

#[cfg(test)]
mod tests {
  // fs_backend
  - list_dir 返回正确结构
  - read_file 处理 UTF-8/非 UTF-8
  - write_file 原子写入

  // git_backend
  - git_status 解析各种状态
  - worktree_list 格式正确

  // pty_manager
  - spawn_pty 启动和退出
  - WebSocket token 验证
  - resize 参数校验

  // project_manager
  - projects.json 序列化/反序列化
  - 最近项目排序逻辑
}

集成测试

验证前后端协作

// Tauri 集成测试框架

关键链路:
- 打开项目 → 文件树加载 → 点击文件
  → 编辑器显示内容
- PTY spawn → WebSocket 连接 →
  输入命令 → 收到输出
- 外部修改文件 → fs 事件 →
  编辑器更新提示
- Worktree 切换 → 全模块重置

工具: tauri-driver (WebDriver)

MVP 测试优先级

优先级 测试内容 理由
P0 必须 PTY spawn + WebSocket 连接 + 数据收发 核心功能，不工作则整个应用没有意义
P0 必须 文件读写 + 编辑器加载/保存 基础编辑功能
P1 重要 fs watcher 事件 + 编辑器冲突处理 AI 修改文件场景的核心体验
P1 重要 Git status + 文件树状态标记 日常使用需要
P2 次要 Worktree 操作 + 项目切换 功能完整性，但非日常高频操作
